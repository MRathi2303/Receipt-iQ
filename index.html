About

Detailed explanation with text and a simple diagram

This section explains the overall system more deeply, including what the frontend does, what the backend does, and where AWS takes over.

Purpose

The app is designed to make receipt submission easy for users while keeping the heavy processing logic in cloud services. The frontend stays clean, and the backend stays minimal.

Frontend role

The frontend focuses on collecting user input, validating the file format and size, and showing a clear status while the receipt moves into the system.

Backend role

The backend accepts the upload request, attaches useful metadata such as the email and category, and places the receipt into cloud storage so the processing pipeline can begin.

AWS role

S3 stores the file, Lambda reacts to the upload event, Textract extracts receipt data, DynamoDB saves the structured result, and SES delivers the final email notification.
